# 归档图片目录

## 📦 目录说明

此目录用于存储被管理员归档的图片文件，**不对外公开访问**。

### 🎯 用途

- **安全合规**: 符合安全审核和监管要求
- **取证备份**: 保留违规或敏感内容用于取证
- **审计追踪**: 完整记录删除历史和操作者信息

### 📂 目录结构

```
archived-images/
├── README.md (本文件)
├── {userId}/
│   ├── {timestamp}_{original_filename}.png
│   └── ...
└── ...
```

### 🔒 安全说明

1. **访问控制**
   - 此目录不在 `public` 目录中
   - Web服务器无法直接访问
   - 仅管理员通过后端API可查看

2. **文件命名**
   - 包含归档时间戳（ISO格式）
   - 保留原始文件名
   - 便于追溯和查找

3. **记录关联**
   - 历史记录中保存 `archivedPath` 字段
   - 可通过用户ID和时间戳定位
   - 支持取证和审计需求

### ⚠️ 注意事项

- **不要手动删除**此目录中的文件
- 归档文件用于合规和取证，需长期保留
- 定期检查磁盘空间使用情况
- 可根据实际需求制定归档清理策略（建议至少保留6个月）

### 📋 清理策略建议

```bash
# 示例：删除6个月前的归档文件（需根据实际合规要求调整）
find ./archived-images -type f -mtime +180 -delete
```
